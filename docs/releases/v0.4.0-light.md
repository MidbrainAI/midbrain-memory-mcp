# midbrain-memory-mcp v0.4.0

v0.4.0 turns MidBrain Memory into an experience-first layer for long-running
coding agents across OpenCode, Claude Code, Codex, and NanoClaw.

The release adds supported setup paths for Codex and NanoClaw, memory-first
project instructions, cross-session continuity checks, and selective procedural
knowledge injection. Agents should start with the right project context without
users having to repeat the same history, decisions, and workflow rules every
session.

Upgrade or install:

```bash
npx -y midbrain-memory-mcp@latest install
```

Then restart OpenCode, Claude Code, Codex, or the relevant NanoClaw group.

## What Changed

- Codex is now supported, including MCP setup, project scoping, capture hooks,
  and bounded tool/activity summaries.
- NanoClaw is now supported through a bundled `/add-midbrain` skill for
  group-scoped setup.
- `check_session_status` gives agents a quick way to detect recent work from
  another session or client.
- `memory_search` can now filter by memory type: `all`, `semantic`, or
  `episodic`.
- Project setup can write MidBrain rules into `AGENTS.md` and `CLAUDE.md`.
- Supported capture hooks can inject relevant procedural knowledge before a
  user turn.
- The package now uses shared client adapters and centralized key resolution
  across MCP tools, hooks, and plugins.

## Trust And Compatibility Notes

Installing a supported client wires local hooks or plugins that contact
`https://memory.midbrain.ai` with the configured API key. By default, those
hooks capture user prompts and assistant messages as episodic memories. Codex
also stores one bounded tool-activity summary per turn.

No intentional breaking change is expected for existing OpenCode or Claude Code
users. Project memory changes still require a client restart because keys are
resolved at startup.

The package was checked with the project release gate, including build, lint,
test suite, docs regression checks, and `npm pack --dry-run`.
