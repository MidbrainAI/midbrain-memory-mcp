# v0.4.6

v0.4.6 makes installer key setup quieter and safer across multiple clients, and
replaces unconditional debug files with bounded, leveled logging.

## TLDR

- Global installs now write one shared MidBrain key by default instead of
  duplicating it into every detected client config directory.
- Interactive multi-client installs can still choose distinct per-client keys,
  and existing distinct per-client keys are preserved.
- Non-interactive installs always stay on the shared global key path and never
  rewrite per-client key files.
- Hook and plugin logging now uses `makeLogger()`, platform log directories,
  log levels, and one-file rotation.
- Debug-level request and payload detail is suppressed by default unless
  `MIDBRAIN_LOG_LEVEL=debug` is set.

## Why

The installer previously copied the same API key into both the global MidBrain
key file and each detected client config directory. That made the normal setup
path noisier than necessary and blurred the distinction between a shared memory
identity and an intentional per-client setup.

This release makes the shared-key path the default. Users who want separate
client identities can still choose that during an interactive multi-client
install, and existing distinct per-client keys remain intact. Non-interactive
installs now consistently use only the global shared key.

The release also tightens capture-hook logging. Earlier hooks wrote
unconditional, unbounded debug logs in home-directory paths. Logging now goes
through one shared leveled logger with platform-appropriate log directories,
bounded rotation, and debug detail disabled by default.

## Changes

- Added an installer prompt for interactive multi-client installs that asks
  whether clients should share one key.
- Changed global install behavior to write only
  `~/.config/midbrain/.midbrain-key` by default.
- Preserved distinct existing per-client key files during interactive installs.
- Fixed non-interactive installs with distinct existing client keys so they
  write only the global key and do not rewrite per-client files.
- Updated installer documentation and project agent rules for the shared-key
  policy.
- Replaced the old debug appender with `makeLogger()`, `logFile()`, and
  `logDir()` helpers.
- Added `MIDBRAIN_LOG_LEVEL`, `MIDBRAIN_LOG_MAX_SIZE`, and
  `MIDBRAIN_LOG_DIR` controls.
- Moved per-client logs into platform-appropriate locations with bounded
  rotation to `<file>.1`.
- Migrated OpenCode, Claude Code, Codex, and `MidbrainApi.storeEpisodic()` call
  sites to the logger object API.

## Compatibility

The MCP tool surface is unchanged. Existing project keys, global keys, and
per-client key files continue to resolve through the same key-resolution chain.

Users with distinct per-client keys keep that setup. Users with the same key
duplicated across multiple client config directories can move to the simpler
global-key setup on their next install.

Log locations changed from legacy home-directory debug files to the
platform-specific MidBrain log directory. Set `MIDBRAIN_LOG_DIR` to override the
location.
