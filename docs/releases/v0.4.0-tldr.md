# v0.4.0 TLDR

MidBrain Memory v0.4.0 is the experience-first release: fewer blank starts,
less repeated context, and agents that carry project history across sessions.

- Adds Codex support.
- Adds NanoClaw support through `/add-midbrain`.
- Adds `check_session_status` so agents can detect recent work from other
  sessions before planning from stale context.
- Adds `memory_type` filtering to `memory_search`.
- Adds memory-first project rules for `AGENTS.md` and `CLAUDE.md`.
- Adds automatic procedural knowledge injection before supported user turns.
- Centralizes client adapters, key resolution, API access, and capture helpers.
- Keeps project keys safer: existing keys are preserved, key files use
  `chmod 600`, and full keys are not printed.

Upgrade:

```bash
npx -y midbrain-memory-mcp@latest install
```

Restart the client afterward.

Trust note: supported installs wire local hooks/plugins that send captured
prompts and assistant messages to `https://memory.midbrain.ai` using the
configured API key. Codex also stores bounded tool-activity summaries. Use
`--no-rules` only to skip instruction-file writes; it does not disable capture
hooks.
