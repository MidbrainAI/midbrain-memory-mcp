# midbrain-memory-mcp v0.4.1

## TLDR

v0.4.1 is a patch release for the OpenCode and NanoClaw setup paths introduced
in `0.4.0`.

- OpenCode no longer exposes internal helper values as plugin exports, fixing
  the `Plugin export is not a function` loader failure.
- OpenCode plugin repair now verifies installed plugin and bundle contents, not
  only a marker file.
- NanoClaw hook setup now uses published `@latest` hook commands so new groups
  can re-resolve the current package on cold start.
- Existing NanoClaw groups pinned to `0.3.2` or package-store hook paths still
  need one migration.
- Codex runtime behavior is unchanged from `0.4.0`; no Codex runtime fix was
  present in the verified release diff.
- Breaking changes: None.

Upgrade or install:

```bash
npx -y midbrain-memory-mcp@latest install
```

Then restart the relevant client or NanoClaw group.

## What Changed

### OpenCode Loader Hotfix

The OpenCode plugin now exports only callable plugin functions: the named
`MidBrainMemoryPlugin` export and a default export that points to the same
plugin.

Internal OpenCode helpers and constants are private again. This fixes the
loader class of failures where OpenCode sees a non-callable export and reports:

```text
Plugin export is not a function
```

The regression test now imports the plugin and verifies every exported value is
callable, that the default export equals the named plugin export, and that the
internal helper exports are not visible.

### OpenCode Freshness Repair

OpenCode stale-plugin detection is stricter. A matching marker file is no
longer enough for the installed plugin to count as fresh.

`OpenCode.isFresh()` now verifies:

- the marker matches the current package marker,
- the installed `midbrain-memory.ts` content matches the repo copy,
- the installed `midbrain-shared.mjs` bundle content matches the repo copy.

This prevents an interrupted or manually reverted repair from being treated as
fresh just because the marker file still matches.

### NanoClaw Auto-Updating Hook Commands

NanoClaw setup now writes Claude capture hooks through the published package:

```text
npx -y midbrain-memory-mcp@latest hook claude user
npx -y midbrain-memory-mcp@latest hook claude assistant
```

The new CLI dispatcher handles those published-package hook commands by
importing the existing Claude Code capture wrappers without starting the MCP
server. Unknown hook commands fail closed with usage text.

The NanoClaw skill and README now tell agents to use these `@latest` hook
commands instead of discovering and writing `/pnpm/.../midbrain-memory-mcp@<version>/...`
paths. New NanoClaw groups should therefore pick up future releases on cold
start after npm resolves `midbrain-memory-mcp@latest`.

## [IMPORTANT NOTE] Existing NanoClaw Groups Need One Migration

This release does not automatically repair existing NanoClaw groups that are
already pinned to `midbrain-memory-mcp@0.3.2` or to a
`/pnpm/.../midbrain-memory-mcp@<version>/...` hook path.

Those groups need one migration to replace the old MCP and hook config
with the new `@latest` commands. After that migration, they should update
normally going forward.

Short migration path:

1. In the NanoClaw directory, identify the target group:

   ```bash
   bash bin/ncl groups list --json
   ```

2. Remove the stale MidBrain MCP server and add it back with `@latest`:

   ```bash
   bash bin/ncl groups config remove-mcp-server --id "$AGENT_GROUP_ID" --name midbrain-memory
   bash bin/ncl groups config add-mcp-server \
     --id "$AGENT_GROUP_ID" \
     --name midbrain-memory \
     --command npx \
     --args '["-y", "midbrain-memory-mcp@latest"]' \
     --env '{"MIDBRAIN_CLIENT":"claude","MIDBRAIN_API_KEY":""}'
   ```

   Fill the API key only in the local NanoClaw config. Do not paste real keys
   into chat, diffs, logs, or support requests.

3. In `data/v2-sessions/<group-id>/.claude-shared/settings.json`, replace old
   MidBrain hook commands with:

   ```text
   MIDBRAIN_API_KEY=<local-key> npx -y midbrain-memory-mcp@latest hook claude user
   MIDBRAIN_API_KEY=<local-key> npx -y midbrain-memory-mcp@latest hook claude assistant
   ```

4. Restart the group, then verify the mounted settings show the new commands:

   ```bash
   grep -E 'midbrain-memory-mcp@latest hook claude (user|assistant)' \
     "data/v2-sessions/${AGENT_GROUP_ID}/.claude-shared/settings.json"
   ```

## Codex

The verified `v0.4.0..HEAD` diff contains no Codex runtime changes under
`plugins/codex`, `shared/clients/codex.mjs`, or `tests/codex-hooks.test.mjs`.

This release leaves Codex behavior unchanged from `0.4.0`. The only
Codex-related files in the release delta are documentation and generated
instruction files that mention existing Codex support.
