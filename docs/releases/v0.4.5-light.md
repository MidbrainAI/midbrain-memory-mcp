# v0.4.5 Light Notes

v0.4.5 makes episodic memory capture more resilient during temporary network or
API outages.

Failed episodic writes are now cached locally and retried after a later
successful write. Cache files are scoped by API key and project, so one account
or workspace cannot flush another account's cached entries.

This release also fixes the concurrent flush ownership path: when multiple
processes try to flush the same cache, only the process that claimed the
processing batch can delete or merge it. Interrupted batches can still recover
on a later flush, and survivor entries plus concurrent live appends are
preserved.

The release includes focused cache, API, and Codex hook tests, plus the full
project check and package dry-run before publish. Breaking changes: none.
