# v0.4.3 Light Notes

v0.4.3 disables automatic procedural-knowledge injection by default for Codex,
Claude Code, and OpenCode hooks.

Users do not need to configure anything to get the safer behavior after
upgrading. Installer-generated configs and hooks do not set
`MIDBRAIN_ENABLE_PK_INJECTION`.

The legacy injection path is still present for compatibility and testing, but
it only runs when `MIDBRAIN_ENABLE_PK_INJECTION=1` is set explicitly.

Episodic capture, explicit memory tools, `MidbrainApi.searchProcedural()`, and
assistant-side scrubbing of old injected PK blocks remain in place.
