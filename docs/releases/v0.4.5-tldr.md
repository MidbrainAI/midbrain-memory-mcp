# v0.4.5 TLDR

- Failed episodic writes are cached locally and retried later.
- Cache files are scoped by API key and project to avoid cross-account flushes.
- Concurrent cache flushes now use explicit processing-batch ownership.
- Interrupted batches, survivor entries, and live appends are preserved.
- Cache, API, and Codex hook tests cover the outage-resilience path.
