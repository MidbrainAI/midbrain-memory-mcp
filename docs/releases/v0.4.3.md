# v0.4.3

v0.4.3 disables automatic procedural-knowledge injection by default in Codex,
Claude Code, and OpenCode hooks while preserving the legacy injection code path
behind an explicit environment opt-in.

## TLDR

- Automatic PK injection is disabled by default.
- Users get the disabled behavior automatically when upgrading; generated
  configs and hooks do not set `MIDBRAIN_ENABLE_PK_INJECTION`.
- The legacy injection path remains available only when
  `MIDBRAIN_ENABLE_PK_INJECTION=1` is set explicitly.
- Episodic capture, explicit memory tools, `MidbrainApi.searchProcedural()`,
  and assistant-side legacy PK scrubbing remain.

## Why

The v0.4.0 line introduced automatic procedural-knowledge injection before
supported user turns. In practice, some procedural memories were stale,
unrelated, or too strong, which could bloat context and steer agents away from
the user's current intent.

v0.4.3 is a narrow disable-only patch. It stops invisible default steering while
keeping the implementation available for explicit compatibility testing and a
future redesigned experience layer.

## Changes

- Codex user hooks store prompts but skip procedural search unless
  `MIDBRAIN_ENABLE_PK_INJECTION=1`.
- Claude Code user hooks store prompts and emit no PK stdout unless explicitly
  opted in with the same flag.
- OpenCode user-message hooks store user text without mutating message parts
  unless the flag is set.
- Assistant capture still scrubs legacy injected PK blocks before storing
  assistant-authored memories.
- Current docs and generated MidBrain rules now describe explicit memory-tool
  recall rather than default automatic PK steering.
- Installer-generated configs and hooks do not set the opt-in flag.

## Compatibility

`MidbrainApi.searchProcedural()` remains available as an internal compatibility
method. The MCP tool surface is unchanged.

If you intentionally need the legacy automatic injection behavior for testing,
set:

```sh
MIDBRAIN_ENABLE_PK_INJECTION=1
```

Do not set this flag in generated user configs unless a user deliberately opts
into the legacy behavior.

## Testing

- Default-off Codex, Claude Code, and OpenCode hook behavior.
- Explicit opt-in Codex, Claude Code, and OpenCode legacy injection behavior.
- Assistant legacy PK scrub behavior.
- Generated docs/rules behavior.
- Full `npm run check` and package dry-run before publish.

## Breaking Changes

None. Automatic PK injection is disabled by default, but capture and explicit
memory tools continue to work.
